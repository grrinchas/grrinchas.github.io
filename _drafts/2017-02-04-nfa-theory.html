In previous tutorial (see {{ POST100.title }}) we talked about DFAs and implemented one in JAVA (see {{ POST101.title }}). Here we will talk about NFA (Nondeterministic Finite Automaton) or NFSM (Nondeterministic Finite State Machine).

{{ page.headers[0].name }}

Just by looking at the names of NFA and DFA, we can see the main difference between them. That is, DFA is a deterministic while NFA is nondeterministic. But what exactly means for automaton to be deterministic?. From {{ POST100_LINK_DEF100 }} we see that given any state and any symbol there is only one transition to the next state. Thus, next state is determined. While NFA has no such constraint.
